# 飞书知识库操作复盘 - 踩坑记录

> 2026-05-07 以机器人身份在"互联网行研"知识库创建文档，耗时 ~25 分钟

## 最终成功路径

1. 从 `~/.openclaw/openclaw.json` 的 `channels.feishu.accounts` 获取 appId + appSecret
2. 调 `auth/v3/tenant_access_token/internal` 获取 TAT
3. 调 `wiki/v2/spaces/{space_id}/nodes` 创建 docx 节点
4. 调 `feishu_update_doc` 填充文档内容

## 踩坑清单（按实际发生顺序）

### 坑 1：lark-cli 找不到（PATH 问题，不是没装）
- **现象**：`lark-cli: command not found`
- **真实情况**：`@larksuite/cli` v1.0.17 已全局安装，位置 `~/.npm-global/bin/lark-cli`
- **根因**：exec 的 PATH 不包含 `~/.npm-global/bin`
- **教训**：先 `npm list -g` 或 `find` 确认是否已安装，别急着重装

### 坑 2：钥匙串里没有 appSecret
`~/.lark-cli/config.json` 指向 keychain，但 keychain 里没有。真实凭证在 `~/.openclaw/openclaw.json` → `channels.feishu.accounts`。

### 坑 3：不知道 wiki_space_id
等用户手动提供。应该先调 `GET /open-apis/wiki/v2/spaces`。

### 坑 4：机器人没有知识库权限
API 返回 `131006 permission denied`。用哪个 bot 的 token 都行，问题是 wiki 空间成员列表里没有这个 bot。需要先加为管理员。

### 坑 5：直接调 Docx API 创建 blocks 失败
裸调 `docx/v1/documents/{id}/blocks/{id}/children` 连续报三个错：`99992402`、`1770001`、`1770029`。应该直接用 `feishu_update_doc`。

### 坑 6：用 `openclaw agent --agent main` 反复调 wiki 工具
`openclaw agent` 命令启动子进程很慢（每次要等 30-60s），来回调了好几次（添加管理员、更新文档内容）。其实拿到凭证后直接用 curl 调 API 更快。

### 坑 7：反复折腾 lark-cli（坑 1 的连锁反应）
同一个问题踩了两遍：
- 用 npx 尝试装 `@larksuite/lark-cli`（包不存在，实际叫 `@larksuite/cli`）
- 跑 `openclaw-lark-tools` 诊断、探索 OAuth、翻 token-store 源码等

本质是坑 1 的连锁反应——lark-cli 就在 `~/.npm-global/bin` 里，只是 PATH 没覆盖到。

### 坑 8：误以为子 agent 不能调用飞书 MCP 工具
- **现象**：webresearcher 调用不了 `feishu_update_doc` 等工具
- **错误归因**：以为是架构限制（只有 main agent 能用）
- **真实原因**：`tools.alsoAllow` 为空
- **已修复**：所有 7 个 agent 都补上了 37 个飞书工具

## 下次快速路径

```bash
export PATH="$HOME/.npm-global/bin:$PATH"

# 1. 拿凭证（任意 bot 都行）
APP_ID=$(jq -r '.channels.feishu.accounts.web_bot.appId' ~/.openclaw/openclaw.json)
APP_SECRET=$(jq -r '.channels.feishu.accounts.web_bot.appSecret' ~/.openclaw/openclaw.json)

# 2. 获取 TAT
TENANT_TOKEN=$(curl -s -X POST "https://open.feishu.cn/open-apis/auth/v3/tenant_access_token/internal" \
  -H "Content-Type: application/json" \
  -d "{\"app_id\":\"${APP_ID}\",\"app_secret\":\"${APP_SECRET}\"}" | jq -r '.tenant_access_token')

# 3. 创建 wiki 节点
curl -s -X POST "https://open.feishu.cn/open-apis/wiki/v2/spaces/${SPACE_ID}/nodes" \
  -H "Authorization: Bearer ${TENANT_TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"parent_token":"", "title":"标题", "obj_type":"docx", "node_type":"origin"}'

# 4. 填充内容 → 直接调 feishu_update_doc（现在所有 agent 都有了）
```

## 关键 API

| 操作 | 方法 | 端点 |
|------|------|------|
| 获取 TAT | POST | `/open-apis/auth/v3/tenant_access_token/internal` |
| 知识库列表 | GET | `/open-apis/wiki/v2/spaces` |
| 创建节点 | POST | `/open-apis/wiki/v2/spaces/{space_id}/nodes` |
| 文档信息 | GET | `/open-apis/docx/v1/documents/{document_id}` |
