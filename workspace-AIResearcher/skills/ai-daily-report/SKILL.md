# AI 行业日报生成 Skill

生成面向银行一线的 AI 行业双轨日报（动态 + 商机），推送至飞书群。

## 何时使用

- 用户说"生成日报"、"出日报"、"AI 日报"、"行业日报"
- 定时任务触发（工作日每日生成）
- 补发历史日报

## 核心原则

1. **双轨并行**：日报A（行业动态）+ 日报B（业务赋能），合并写入同一文档
2. **地域聚焦**：浙江为主，长三角（上海/江苏/安徽）为辅，其他只看重大事件
3. **去重机制**：已报过且无新进展的事件，次日不再输出
4. **业务转化**：每条商机必须落到六要素（客户/信号/话题/方案/额度/避雷）
5. **窗口期标注**：P0（<14天必访）/ P1（1-3月储备）/ 预警
6. **Markdown 格式**：输出使用 Markdown 格式（标题、列表、表格等）
7. **知识库写入**：以机器人身份在「AI 行研」知识库创建节点，标题：AI-daily-YYYYMMDD
8. **日报周报分离**：日报只出日报内容，周报由 ai-weekly-report skill 独立负责

## Token 优化原则

- **web_fetch 用 text 模式**（`extractMode: "text"`），比 markdown 模式更精简
- **maxChars=6000**，够提取新闻标题和摘要，不需要更大
- **表格只查 A 列**（`28609d!A:A`）去重，不查全表
- **lark-cli 命令按预定义参数执行**，不要反复试错
- **合并同源**：财联社电报 + 同花顺已覆盖大部分 A 股动态，不重复抓
- **正文点入按需**：只在发现重要线索时点入正文（maxChars=4000）

## 工作流

### 第一步：信息抓取（并行）

**必抓源（每个工作日，4个）：**

```
1. 同花顺首页 → web_fetch https://www.10jqka.com.cn/  (extractMode="text", maxChars=6000)
2. 财联社电报 → web_fetch https://www.cls.cn/telegraph  (extractMode="text", maxChars=6000)
3. 新华网科技 → web_fetch https://www.xinhuanet.com/tech/  (extractMode="text", maxChars=6000)
4. 经济观察网 → web_fetch https://www.eeo.com.cn/  (extractMode="text", maxChars=6000)
```

**补充源（按需，时间充裕时抓）：**

```
5. 36氪 AI 频道 → web_fetch https://www.36kr.com/information/AI/  (extractMode="text", maxChars=6000)
```

**抓取规则：**
- 4个必抓源并行执行
- 如果 web_fetch 返回空白/JS 渲染失败，跳过该源，不要重试
- 从新华网等源发现重要文章时，点入正文抓取（extractMode="text", maxChars=4000）
- 只关注 **过去24小时内** 发生的事件

### 第二步：去重检查

**使用「商机挖掘」电子表格去重（只查客户名称列 A）：**
```
spreadsheet_token: FwBOsiErhh5U9Qtr5pvclx9Zncl
sheet_id: 28609d
```

查询命令（只查 A 列）：
```
lark-cli sheets +read --spreadsheet-token FwBOsiErhh5U9Qtr5pvclx9Zncl --range "28609d!A:A"
```

**去重逻辑（智能模糊匹配，必须遵守）：**
1. 提取新商机企业名称的**核心简称**（去掉括号、股票代码、地域后缀等修饰），如：
   - "曹操出行（杭州）" → 核心简称 "曹操出行"
   - "中控技术（杭州，688777）" → 核心简称 "中控技术"
   - "浙江史河科技（杭州）" → 核心简称 "史河科技"
2. 将核心简称与表格A列中已有名称做**包含匹配**：如果A列已有名称包含该核心简称，或核心简称包含A列已有名称的核心部分，视为同一企业
3. **如果已存在** → 原地更新该行（`lark-cli sheets +write --range "28609d!A{行号}:J{行号}"`），**保持A列原名称不变**，只更新其他列
4. **如果不存在** → 追加新行（`lark-cli sheets +append`）
5. 长期趋势/行业常识 → 直接跳过，不写入表格

⚠️ **严禁**：更新行时修改A列客户名称（包括加后缀、改格式等），必须保持原名称不变

**如果电子表格不可用**：
回退到读取 `skills/ai-daily-report/references/event-log.json` 作为本地缓存。

### 第三步：筛选与分类

从抓取的信息中筛选出符合以下标准的条目：

**入选标准（满足任一即可）：**
- 政策发布/修订（影响资金流向或授信风险）
- 企业融资/上市/定增公告
- 产能建设/设备搬入/产线投产
- 订单/定点公告
- 技术突破/产品发布
- 价格变动（AI 芯片、算力资源、数据服务等，影响产业链成本）
- 风险信号（订单下修、减持、诉讼等）

**排除标准：**
- 行业常识科普
- 长期趋势分析（无具体事件）
- 与 AI 行业（大模型、AI 应用、AI 基础设施、AI 芯片、机器人、自动驾驶、智能驾驶等）无关的内容
- 纯股市行情分析（无产业事件支撑）

**地域标签：**
- `[浙江]` / `[杭州]` / `[宁波]` / `[绍兴]` 等
- `[上海]` / `[江苏]` / `[南京]` / `[苏州]` / `[南通]` 等
- `[安徽]` / `[其他]` / `[全球]`

**重要性评级：**
- ★★★★★：国家级政策 / 百亿级投资 / 行业拐点
- ★★★★：省级政策 / 十亿级融资 / 重大订单
- ★★★：企业级公告 / 产能变动 / 价格变动
- ★★：一般动态
- ★：值得关注但影响有限

### 第五步：生成日报B（业务赋能）- Markdown 格式

**地域范围：仅聚焦浙江本地企业**（杭州/宁波/温州/绍兴/嘉兴/湖州/金华/台州/丽水/衢州/舟山）。日报A中非浙江的事件不转化为商机。浙江本地事件优先转化为业务机会。

```markdown
# 📌 AI 行业日报·商机 | YYYY-MM-DD 周X

## 🎯 P0 必访（窗口期<14天）

### 【客户X】企业名称（城市，股票代码）
- **信号**：具体事件 + 日期
- **话题**：具体拜访开场白，引用最新公开动态
- **方案**：
  - 产品1（触发条件）
  - 产品2（触发条件）
- **额度**：预计金额区间
- **联系**：对接部门/岗位
- **避雷**：注意事项

## 📋 P1 储备（窗口期1-3月）

### 【客户X】企业名称（城市）
- **信号**：事件
- **话题**：拜访话题
- **方案**：产品方案
- **额度**：预计区间
- **联系**：对接部门
- **避雷**：注意事项

## ⚠️ 预警

- 风险信号：影响分析
- 风险信号：影响分析

## 📊 今日商机汇总

| 类别 | 数量 |
|------|------|
| P0必访 | X家 |
| P1储备 | X家 |
| 预警 | X条 |
| 预计可介入金额 | 流贷/订单融资约X亿 | 信用证约X亿 | 资金监管X亿+ |

## 💡 本周行动建议

1. 行动项1
2. 行动项2
3. 行动项3
4. 行动项4
```

**商机转化规则：**

| 产业事件 | 对应产品 | 窗口期 |
|---------|---------|-------|
| IPO/定增公告 | 募集资金专户、资金监管 | 公告后72h |
| 融资到账 | 专户设立、流贷储备 | 到账后30天 |
| 设备搬入/产线投产 | 进口信用证、设备融资租赁 | 设备到港前7天 |
| 订单/定点公告 | 订单融资、原材料信用证 | 公告后14天 |
| 产能扩张公告 | 流贷追加、项目贷款 | 公告后1-3月 |
| 价格变动（涨价） | 供应链金融、库存融资 | 涨价后7天 |
| 价格变动（跌价） | 预警、授信重估 | 立即 |
| 政策补贴落地 | 配套融资、专户管理 | 政策发布后30天 |

**产品库参考：**
- 流动资金贷款、固定资产贷款、项目贷款
- 进口/出口信用证、保函、银行承兑汇票
- 订单融资、供应链金融、应收账款保理
- 募集资金专户监管、跨境资金池
- 汇率远期锁汇、期权
- 超募资金理财、结构性存款
- 股票质押（慎用，标注风险）

### 第六步：更新/追加商机到表格

日报B中的商机数据（P0/P1/预警）写入「商机挖掘」电子表格。

**🔴 关键规则（必须严格遵守）：**
- **只写入浙江本地企业**（杭州/宁波/温州/绍兴/嘉兴/湖州/金华/台州/丽水/衢州/舟山），非浙江企业一律不写入表格
- **写入前必须去重**：先读取 A 列检查客户名称是否已存在，存在则更新该行，不存在才追加
- **禁止重复写入同一企业**：如果同一家企业多次出现在日报中，只保留最新的一条记录

**表格信息：**
```
spreadsheet_token: FwBOsiErhh5U9Qtr5pvclx9Zncl
range: 28609d!A:J
```

**写入流程：**
1. 先读取 A 列去重：`lark-cli sheets +read --spreadsheet-token FwBOsiErhh5U9Qtr5pvclx9Zncl --range "28609d!A:A"`
2. 对每个商机客户：
   - 如果 A 列已存在相同客户名称 → 用 `lark-cli sheets +write` 原地更新整行，**保持A列原名称不变**
   - 如果不存在 → 用 `lark-cli sheets +append` 追加新行
3. 写入后验证：读取 A 列确认无重复

⚠️ **lark-cli 限制**：不支持真正的"删除行"操作（只能清空内容）。如果之前操作产生了空行，追加新商机时会跳过空行追加到更后面，导致中间有大量空行。后续追加新商机时，如果发现中间有空行，应先将空行之后的数据整体前移覆盖空行，再追加到末尾。

**更新整行示例：**
```
lark-cli sheets +write --spreadsheet-token FwBOsiErhh5U9Qtr5pvclx9Zncl --range "28609d!A{行号}:J{行号}" --values '[[客户名称, 行业/领域, 触发信号, 优先级, 推荐方案, 预计金额, 联系人, 状态, 创建日期, 备注]]'
```

**追加新行示例：**
```
lark-cli sheets +append --spreadsheet-token FwBOsiErhh5U9Qtr5pvclx9Zncl --range "28609d!A:J" --values '[[...],[...]]'
```

**字段顺序**：客户名称、行业/领域、触发信号、优先级、推荐方案、预计金额、联系人、状态、创建日期、备注

**状态说明**：
- 新商机：`状态: "待联系"`
- 已有商机有新进展：更新 `触发信号`，`状态: "待联系"`
- 商机跟进中：`状态: "储备"`
- 风险信号：`状态: "监控"`

每条商机一行，P0/P1商机和预警都要写入。非浙江企业（上海/江苏/安徽/北京/深圳等）不写入表格。

### 第七步：写入知识库

**重要：每次生成都覆盖当前同名文件（AI-daily-YYYYMMDD），不要有重复日期的文档。**

**🔴 关键规则（必须严格遵守）：**
- 文档必须创建在知识库**根目录**（`parent_node_token` 为空字符串），**不能**创建在「首页」或其他节点下面
- 必须使用**机器人身份**（`--as bot`）创建，创建者显示为机器人
- 空间名称是 `AI行研`（没有空格），不是 `AI 行研`

**步骤：**

1. **列出知识库根目录节点，查找是否已有同名文档**：
   ```bash
   lark-cli wiki nodes list --params '{"space_id":"7630717889183534041","page_size":50}'
   ```
   从返回结果中搜索 title 为 `AI-daily-YYYYMMDD` 且 `parent_node_token` 为 `""`（根目录）的节点，提取 `obj_token`。

2. **如果找到同名文档（根目录下）**：
   - 提取 obj_token
   - 使用 `lark-cli docs +update --doc <obj_token> --as bot --mode overwrite --markdown '<内容>'` 覆盖内容
   - 输出文档链接：`https://www.feishu.cn/wiki/<node_token>`

3. **如果未找到同名文档**：
   - 使用以下命令在**根目录**以**机器人身份**创建：
     ```bash
     lark-cli wiki +node-create --as bot \
       --space-id "7630717889183534041" \
       --title "AI-daily-YYYYMMDD" \
       --obj-type "docx"
     ```
   - 从返回结果提取 `obj_token`（用于内容更新）和 `node_token`（用于 URL）
   - 使用 `lark-cli docs +update --doc <obj_token> --as bot --mode overwrite --markdown '<内容>'` 写入内容
   - 输出文档链接：`https://www.feishu.cn/wiki/<node_token>`

**验证步骤（创建后必须执行）：**
```bash
# 确认文档在根目录且创建者是机器人
lark-cli wiki nodes list --params '{"space_id":"7630717889183534041"}' | \
  jq '.data.items[] | select(.title=="AI-daily-YYYYMMDD") | {node_token, parent_node_token, creator}'
# parent_node_token 应为 ""，creator 应为机器人 openid（ou_e18561928e9f46412d45ca124a960fee）
```

**注意**：
- 日报A和日报B合并写入同一文档，用 `---` 分隔
- 使用 Markdown 格式（标题、列表、表格等）
- 标题格式：AI-daily-YYYYMMDD（如 AI-daily-20260428）
- wiki URL 必须使用 node_token 而不是 obj_token
- **禁止**使用 `feishu_wiki_space_node` 工具（该工具可能将文档创建在首页下），必须使用 `lark-cli wiki +node-create --as bot --space-id` 命令

### 第八步：推送至群聊（概要 + 链接）

**🔴 关键规则：推送到群聊的内容是概要和链接，不是全文！**

推送至群聊的消息格式如下：

```markdown
📌 **AI 行业日报 | 2026-04-28 周二**

🔥 今日重点：
1. 英伟达创历史新高（5.26万亿美元）
2. 发改委禁止外资收购Manus项目
3. 台积电2nm首年产出提升约45%

📍 浙江/长三角：史河科技、曹操出行、同花顺等 X 条
📊 商机：P0 X家 | P1 X家 | 预警 X条

📄 日报全文：<https://www.feishu.cn/wiki/XXXX>
📊 商机表格：<https://xxx.feishu.cn/sheets/XXXX>
```

**推送内容要求：**
- 只输出 3-5 条今日重点标题（一句话）
- 商机汇总数字（P0/P1/预警数量）
- 日报文档链接（wiki URL）
- 商机挖掘表格链接（sheets URL）
- **不要**输出日报全文内容

**获取表格链接：**
```bash
# 商机挖掘表格链接
lark-cli sheets +read --spreadsheet-token FwBOsiErhh5U9Qtr5pvclx9Zncl --range "28609d!A1" 2>&1
# 表格 URL 格式：https://<domain>.feishu.cn/sheets/<spreadsheet_token>
```

### 信息源全部失败

如果所有 web_fetch 都失败或返回空白：
1. 不要编造内容
2. 告知用户信息源抓取失败，建议手动提供素材
3. 或基于已知信息生成最小化版本并标注"信息有限"
